# API Yatube

## Yatube социальная сеть для публикации личных дневников

Yatube социальная сеть для  постов, с возможностью комментировать посты других участников а также подписываться на авторов.

## Особенности API Yatube

- Эндпоинты API Yatube доступны всем пользователям для чтения постов и комментариев к ним.
- API Yatube предоставляет возможность для входа аутентифицированным пользователям через предварительно полученный Token.
- Возможность подписаться на других авторов есть только у аутентифицированных пользователей.
- Аутентифицированным пользователем предоставлена возможность просматривать свои подписки с поиском среди них необходимого автора.
- Публиковать посты могут также только аутентифицированные пользователи.
- Изменять или удалять посты и комментари может только автор контента.

## Установка API Yatube

### Как запустить проект

- Клонировать репозиторий и перейти в него в командной строке:

```
git clone git@github.com:InsaraOK/api_final_yatube.git
```

```
cd api_final_yatube
```

- Cоздать и активировать виртуальное окружение:

```
py -m venv venv
```

```
source venv/Scripts/activate
```

```
python -m pip install --upgrade pip
```

- Установить зависимости из файла requirements.txt:

```
pip install -r requirements.txt
```

- Выполнить миграции:

```
python manage.py migrate
```

- Запустить проект:

```
python manage.py runserver
```

## Примеры

### Некоторые примеры запросов к API Yatube

- Получение публикаций:
Получить список всех публикаций Get-запросом:

```
http://127.0.0.1:8000/api/v1/posts/
```

При указании в запросе параметров limit (целое число) - количество публикаций на страницу и offset (целое число) - номер страницы после которой начинать выдачу GET-запрос вернёт два объекта, с третьего по четвертый при их наличии.

```
http://127.0.0.1:8000/api/v1/posts/?limit=2&offset=2 
```

- Получить JWT-токен:

```
http://127.0.0.1:8000/api/v1/jwt/create/
```

POST- запрос вернет 2 Токена при вводе своего имени и пароля в тело запроса:
"refresh": для восстановления утраченного Токена
"access": основной Токен.

- Подписка на автора:

```
http://127.0.0.1:8000/api/v1/follow/
```

Подписка пользователя от имени которого сделан запрос на пользователя переданного в теле запроса. POST - запросе необходимо указать в теле запроса поле "following": "имя автора"
> Note: имя автора должно точно совпадать с указанным на его странице.
